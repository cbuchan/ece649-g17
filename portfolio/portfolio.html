Portfolio Overview

  • Design
    • Architecture Diagram - The architecture diagram describes the objects present in the system, the replication of distributed objects, and the communication channels between objects.
    • Use Cases - The use case diagram describes the ways that agents in the system can interact with the elevator and shows relationships between these methods of interaction.
    • Scenarios and Sequence Diagrams - A compilation of use case scenarios and corresponding sequence diagrams.
    • Requirements I - System Object Descriptions and Message Dictionary - A list of the sensors and actuators in the system and a message dictionary describing the messages that are required for the various controllers.
    • Requirements II - Distributed Controller Requirements - Detailed descriptions for each of the system components.
  • Traceability
    • Sequence Diagrams to Requirements Traceability - Form ensuring forward traceability from sequence diagrams to requirements.
    • Requirements to Contstraints Traceability - Form ensuring traceability from requirements to constraints.
    • Statecharts to Code Traceability - This document shows where each transition described in the controller state-charts is implemented in the code.
  • Implementation
    • Elevator Control Package - This package implements the elevator framework, payloads, controllers, and modules.
  • Test
    • Unit Test Log - This log documents the unit tests performed on each individual controller.
    • Unit Test Summary File - Searchable list of unit-test files
    • Integration Test Log - This log documents the integration testing results.
    • Integration Test Summary File - List of integration test files
